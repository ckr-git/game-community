# 游戏社区交流平台 - 启动说明

## 1. 环境准备

### 必需软件
- **JDK 17** 或更高版本
- **Maven 3.6+**
- **MySQL 8.0+**
- **Redis 6.0+**（可选，推荐安装以支持缓存）
- **Node.js 16+**（前端开发）
- **IDEA** 或其他Java IDE

## 2. 数据库配置

### 创建数据库
```sql
-- 方式1：直接在MySQL中执行database.sql文件
mysql -u root -p < database.sql

-- 方式2：使用MySQL客户端工具导入database.sql
```

### 修改数据库连接
打开 `src/main/resources/application.yml`，修改数据库配置：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/game_community?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root          # 改成你的MySQL用户名
    password: root          # 改成你的MySQL密码
```

## 3. 启动Redis（可选但推荐）

### Windows
```bash
# 启动Redis服务
redis-server.exe redis.windows.conf
```

### Linux/Mac
```bash
redis-server
```

如果不启动Redis，项目仍可运行，但推荐系统的缓存功能将不可用。

## 4. 启动后端

### 方式1：使用Maven命令
```bash
# 在项目根目录执行
mvn spring-boot:run
```

### 方式2：使用IDEA
1. 用IDEA打开项目
2. 等待Maven依赖下载完成
3. 找到 `CommunityPlatformApplication.java`
4. 右键选择 `Run 'CommunityPlatformApplication'`

### 验证启动成功
- 看到控制台输出：`游戏社区交流平台启动成功！`
- 后端API地址：http://localhost:8080/api

## 5. 启动前端

### 安装依赖
```bash
cd frontend
npm install
```

### 启动开发服务器
```bash
npm run dev
```

### 访问前端
打开浏览器访问：http://localhost:5173

## 6. 测试功能

### 注册测试账号
1. 访问 http://localhost:5173/register
2. 填写注册信息：
   - 用户名：testuser
   - 密码：123456
   - 昵称：测试用户
   - 邮箱：test@example.com

### 登录测试
1. 使用注册的账号登录
2. 进入系统后可以：
   - 浏览游戏列表
   - 查看游戏详情
   - 访问聊天室（WebSocket功能）
   - 浏览论坛帖子

### 测试管理员功能
1. 在数据库中将用户的role字段改为2
   ```sql
   UPDATE user SET role = 2 WHERE username = 'testuser';
   ```
2. 重新登录后可访问管理后台：http://localhost:5173/admin

## 7. 常见问题

### Q1: 启动时提示数据库连接失败
**A:** 检查MySQL服务是否启动，数据库名、用户名、密码是否正确。

### Q2: Maven依赖下载很慢
**A:** 配置Maven国内镜像，编辑 `~/.m2/settings.xml`：
```xml
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <name>Aliyun Maven</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

### Q3: WebSocket连接失败
**A:** 
- 确保后端已正常启动
- 检查浏览器控制台是否有跨域错误
- WebSocket地址格式：`ws://localhost:8080/api/websocket/chat/{roomId}/{userId}`

### Q4: 前端npm install失败
**A:** 
- 切换npm镜像源：`npm config set registry https://registry.npmmirror.com`
- 清除缓存：`npm cache clean --force`

### Q5: Redis连接失败
**A:** 
- 如果不需要缓存功能，可以暂时忽略
- 或安装并启动Redis服务

## 8. 开发建议

### 后端开发
- 使用IDEA的热部署插件，修改代码后自动重启
- 查看日志：控制台会输出所有SQL语句和请求日志

### 前端开发
- Vite提供热更新，保存代码后浏览器自动刷新
- 使用浏览器开发者工具调试API请求
- Vue Devtools插件可以调试Vue组件

## 9. 项目打包

### 后端打包
```bash
mvn clean package
# 打包后的jar文件在 target/ 目录下
```

### 前端打包
```bash
cd frontend
npm run build
# 打包后的文件在 dist/ 目录下
```

## 10. 项目部署

### 后端部署
```bash
# 运行打包好的jar文件
java -jar community-platform-1.0.0.jar
```

### 前端部署
将 `frontend/dist/` 目录下的文件部署到Nginx或其他Web服务器。

## 技术支持

如遇到问题，请检查：
1. 所有环境是否正确安装
2. 配置文件是否正确修改
3. 端口是否被占用（8080, 5173, 3306, 6379）
4. 日志输出的错误信息
